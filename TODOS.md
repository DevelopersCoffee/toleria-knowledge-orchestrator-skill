## Phase 1: Core Read/Write Skills

### [ ] vault.read skill
- **What:** Read note from vault, parse frontmatter + content
- **Input:** file path
- **Output:** {frontmatter, content}
- **Tests:** valid file, missing file, empty file, malformed YAML
- **Effort:** S (human: ~2h / CC: ~15min)

### [ ] vault.write skill
- **What:** Write note atomically (write to .tmp, rename on success)
- **Input:** path, content, frontmatter
- **Output:** success/error
- **Validation:** schema check before write
- **Tests:** valid write, disk full, permission denied, concurrent writes (atomic rename)
- **Effort:** M (human: ~3h / CC: ~30min)

---

## Phase 2: Knowledge Extraction & Linking

### [ ] knowledge.extract skill
- **What:** Parse text/diff, extract decisions + patterns + stack
- **Input:** text or diff
- **Output:** [{type, title, content, tags}]
- **Validation:** pre-write schema check
- **Tests:** happy path, empty input, malformed input, nil input
- **Effort:** M (human: ~3h / CC: ~30min)

### [ ] knowledge.link skill
- **What:** Create wikilinks [[project]], [[stack]], [[related decision]]
- **Input:** entity_id, project, stack
- **Output:** updated frontmatter with links
- **Error handling:** missing project → skip link, log warning
- **Tests:** valid link, missing project, circular link
- **Effort:** S (human: ~2h / CC: ~20min)

---

## Phase 3: Inbox Processing

### [ ] inbox.process skill
- **What:** Classify INBOX notes → normalize → move → link
- **Process:** classify → convert to schema → move to folder → vault.link → vault.validate
- **Error handling:** classification fails → move to INBOX_INVALID
- **Tests:** valid note → moved, classify fails → INBOX_INVALID, schema mismatch → INVALID
- **Effort:** M (human: ~3h / CC: ~40min)

### [ ] utilities (shared)
- **What:** Extract yaml-parse, link-gen, path-norm to .sh modules
- **Location:** utils/yaml.sh, utils/link.sh, utils/path.sh
- **Tests:** each utility tested independently
- **Effort:** S (human: ~1.5h / CC: ~20min)

---

## Phase 4: Validation & Indexing

### [ ] vault.validate skill
- **What:** Schema check + orphan detection + consistency
- **Checks:**
  - Valid JSON frontmatter
  - Required fields: id, type, created_at, updated_at
  - Links resolve to existing notes
  - No duplicate ids
  - project + stack tags present
- **Output:** {violations: [{path, issue, severity}], valid_count, invalid_count}
- **Tests:** valid vault, missing field, broken link, duplicate id
- **Effort:** M (human: ~3h / CC: ~40min)

### [ ] INDEX files (generated)
- **What:** decisions.json, patterns.json, projects.json (fast lookup, no vault scans)
- **Generated by:** vault.write (on every write)
- **Format:**
  ```json
  {
    "decisions": [
      {id, title, stack, project, created_at}
    ]
  }
  ```
- **Tests:** index updates on write, index reflects deletes
- **Effort:** S (human: ~2h / CC: ~25min)
